# ai-clone-shield (배포 코드 복제 방지)

사용자가 "내 프로젝트(사이트) 못 베끼게 해줘" 같은 요청을 하면, **그 프로젝트 폴더에서** 이 스킬을 따라 보호를 적용한다. 이 도구는 의존성 0의 Node CLI(`bin/aishield.js`)이며, Claude는 이 CLI를 호출하는 오케스트레이터로 동작한다.

## 0. 가장 먼저: 정직한 기대치 (반드시 사용자에게 먼저 전달)
복제를 **불가능하게 만들 수 없다.** 공개 웹은 브라우저가 코드를 받아야 렌더되기 때문이다. 이 스킬의 목표는 이것이다:
**복제 비용을 높이고 + 그대로 베끼면 워터마크로 추적되게 하고 + 법적 증거를 남긴다.** 보증이 아니라 억지력과 기록이다. 과장하지 않는다.

## 1. 절차
1. **감지**: `node bin/aishield.js detect` (설치 시 `aishield detect`). 스택·빌드 산출물을 확인.
2. **설정**: `aishield init` → `aishield.config.json` 의 `owner.name/domain/contact` 를 사용자에게 받아 채운다. 이 값이 워터마크·법적 고지의 식별자이므로 비면 식별력이 약하다고 알린다.
3. **빌드**: 사용자 프로젝트를 빌드한다(산출물이 있어야 후처리 가능). 빌드 디렉토리가 안 잡히면 config의 `buildDir` 를 지정.
4. **적용**: `aishield apply` (기본 balanced). 강하게는 `--preset max`, 미리보기는 `--dry`.
5. **검증**: `aishield verify` (가능하면 `--url <배포 미리보기 URL>` 로 실제 렌더까지). **깨지면 즉시 `aishield restore`.**
6. **배포**: 보호된 산출물을 배포.

## 2. 보호 레이어 (효과는 냉정하게 전달)
| | 레이어 | 효과 |
|---|---|---|
| A | 코드 경화(소스맵 제거·난독화 옵션) | 구조 복제 **중** (난독화는 LLM 디오퍼로 우회 가능) |
| B | 로직 격리 진단(노출 시크릿 스캔·권고) | 로직 복제 **상** (서버 이전이 근본 방어) |
| C | 워터마크 + 무결성(제로폭 분산·메타·해시) | 추적·증거 **상** / 변조차단 **중** |
| D | AI 고지(주석·meta·sr-only·llms.txt) | **불확실** (모델 의존, 보조) |
| E | 법적(robots AI봇차단·TDM·독점 LICENSE·무결성) | 소송 기반 **상** (관할권별 차이) |
| F | 기능 무손상 게이트 | 적용이 앱을 깼는지 검증 |
| G | 공격적 억제(가이드) | 기본 **OFF**, SEO 타격·법 회색지대 |

## 3. 복제를 발견했을 때
- `aishield watermark-check --url <의심 URL>` → 워터마크 ID가 내 레지스트리와 일치하면 그 페이지는 내 원본에서 그대로 복제된 것.
- `aishield evidence-pack` → 워터마크·무결성 해시·git 이력·DMCA 초안을 한 폴더로 묶음.

## 4. 항상 지킬 것
- 적용 후 **반드시 `verify`**. 실패하면 `restore`로 되돌리고 원인부터 본다(기능 무손상이 1순위 요구).
- 각 레이어의 효과 라벨(중/상/불확실)을 **그대로** 사용자에게 전달한다. "이제 못 베낀다"고 단정하지 않는다.
- 워터마크는 "그대로 긁어 붙이는 복제" 추적용이다. AI에게 "완전히 새로 만들어줘"를 시킨 재작성 복제에는 전달되지 않는다고 정직하게 말한다.
- 윤리: AI 고지문은 저작권 고지·복제 자제 요청 수준의 비공격적 문구만 생성한다(`ETHICS.md`). 공격형 프롬프트 인젝션은 만들지 않는다.
