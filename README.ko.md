# ai-clone-shield

[English](./README.md) | **한국어**

> 공개 배포된 웹 프로젝트를, **기능은 그대로 둔 채** AI(바이브코딩 에이전트 등)와 사람이 구조·로직을 통째로 베끼기 어렵게 다듬어주는 오픈소스 스킬팩.

자기 프로젝트 폴더에서 한 번 실행하면, 빌드 산출물에 워터마크·AI 고지·법적 헤더를 박고, 적용 후 기능이 안 깨졌는지까지 검증해 줍니다. 나중에 복제가 의심되면 그 사이트에서 내 워터마크를 추출해 입증하고, 분쟁 증거 묶음을 만들 수 있습니다.

스택을 가리지 않습니다(Next.js, Vite, CRA, 정적 HTML, 그리고 어떤 빌드 산출물이든 후처리).

> **가장 쉬운 방법:** 자기 프로젝트 폴더에서 [이 프롬프트](./USE-WITH-AI.md)를 AI 코딩 도구(Claude Code, Cursor 등)에 복붙하면 알아서 설치·점검·적용해줍니다. 명령어 외울 필요 없어요.

---

## 왜 만들었나

바이브코딩 시대에는 누구나 배포된 사이트 주소를 AI에 던지고 "이거랑 똑같이 만들어줘"라고 시킬 수 있게 됐습니다. 공개 사이트는 화면을 그리려면 HTML·CSS·JS를 브라우저로 다 내려보내야 해서, 사람이든 AI든 그대로 읽고 복제할 수 있습니다. 정작 만든 사람이 기댈 수단은 거의 없습니다.

ai-clone-shield는 복제를 불가능하게 만든다고 말하지 않습니다(불가능하니까요. 아래 "정직한 한계"를 보세요). 대신 복제 비용을 올리고, 지우기 어려운 워터마크로 노골적인 복제본을 추적할 수 있게 하고, 법적 증거를 남깁니다. 보증이 아니라 억지력과 기록입니다. 이 도구 전체가 그 점에 정직하도록 설계됐습니다.

---

## 먼저, 정직하게 (이게 이 도구의 정체성입니다)

**복제를 불가능하게 만들지 못합니다.** 공개 웹은 브라우저가 화면을 그리려면 HTML·CSS·JS를 클라이언트로 내려보내야 하므로, "아예 못 베끼게"는 물리적으로 불가능합니다.

이 도구가 실제로 하는 일은 세 가지입니다.

1. **복제 비용을 높입니다.** 소스맵 제거, 핵심 로직 서버 이전 권고, 난독화 옵션.
2. **그대로 베끼면 걸리게 합니다.** 지우기 어려운 다중 분산 워터마크 + 추적 ID.
3. **법적 증거를 남깁니다.** 독점 라이선스, TDM opt-out, 무결성 해시, 타임스탬프, DMCA 초안.

보증이 아니라 **억지력과 기록**입니다. 이 README는 각 기능의 효과를 과장 없이 "상/중/불확실"로 라벨링합니다.

## 빠른 시작

```bash
# 1) 설치 (둘 중 하나)
npx ai-clone-shield <명령>           # 설치 없이
npm i -g ai-clone-shield             # 전역 설치 후 aishield <명령>

# 2) 내 프로젝트 폴더에서
aishield detect                      # 스택·빌드 산출물 감지
aishield init                        # aishield.config.json 생성 → owner 정보 입력
npm run build                        # 프로젝트 빌드(산출물 필요)
aishield apply                       # 보호 적용(기본 balanced)
aishield verify                      # 기능 무손상 확인 (깨지면 aishield restore)
# 그리고 배포
```

`aishield.config.json` 에서 `owner.name / domain / contact` 를 채우세요. 이 값이 워터마크와 법적 고지의 식별자입니다(비면 추적 식별력이 약해집니다).

## 명령

| 명령 | 설명 |
|---|---|
| `aishield detect` | 스택·빌드 산출물 디렉토리 감지 |
| `aishield init` | 설정 파일 생성(`--preset light\|balanced\|max`) |
| `aishield apply` | 보호 레이어 적용(`--dry` 미리보기, `--preset`) |
| `aishield verify` | 적용 후 기능 무손상 검증(`--url <URL>` 시 실제 렌더 검사) |
| `aishield watermark-check <파일>` | 의심 사이트에서 내 워터마크 추출·대조(`--url <URL>`) |
| `aishield evidence-pack` | 복제 분쟁용 증거 번들 생성 |
| `aishield restore` | 마지막 백업으로 원복(`--from <경로>`) |

## 보호 레이어

프리셋(`light` / `balanced` / `max`)으로 켜고 끕니다. 효과는 냉정하게 표기합니다.

| | 레이어 | 하는 일 | 효과 | 한계 |
|---|---|---|---|---|
| **A** | 코드 경화 | 소스맵 제거 + sourceMappingURL 제거 + 난독화(옵션) | 구조 복제 **중** | 난독화는 2025 LLM 디오퍼스케이터로 우회 가능. 소스맵 제거가 더 확실 |
| **B** | 로직 격리 진단 | 클라 번들의 노출 시크릿·위험 패턴 스캔 + 서버 이전 권고 | 로직 복제 **상** | 자동 이전은 불가(코드 구조 변경). 진단·권고만 |
| **C** | 워터마크 + 무결성 | 제로폭 워터마크 다중 분산 + 메타 + 콘솔 배너 + 무결성 해시 + 레지스트리 | 추적 **상** / 변조차단 **중** | 재작성("새로 만들어줘") 복제엔 증발. 그대로 베끼는 복제 추적용 |
| **D** | AI 고지 | HTML 주석 + meta(ai-policy/robots) + 화면에 안 보이는 sr-only 고지 + `llms.txt` | **불확실** | 모델 의존(연구상 성공률 편차 큼), 사람 복제엔 무효한 보조 레이어 |
| **E** | 법적 증거팩 | robots(AI봇 차단·일반 크롤러 허용) + `.well-known/tdmrep.json` + 독점 LICENSE + 무결성 매니페스트 | 소송 기반 **상** | 미국은 등록 전제, TDM opt-out은 EU 내 효력, AI학습금지 조항은 판례 미확립 |
| **F** | 기능 무손상 게이트 | 적용 후 주입 태그 닫힘·자산 참조·JS 구문 정적 검증 | 회귀 방지 | 정적 검증. 깊은 검증은 `verify --url` |
| **G** | 공격적 억제 | 서버 봇 차단·PoW(Anubis) 설정 **가이드 생성** | 기본 **OFF** | SEO 타격(검색 크롤러도 차단), UA 위조로 우회. 옵트인(`aggressiveOptIn`)만 |

## 복제를 발견했을 때

```bash
# 의심 사이트에서 내 워터마크를 추출해 대조
aishield watermark-check --url https://copycat.example/

# 일치하면: 분쟁 증거 번들 생성(워터마크·해시·git·DMCA 초안)
aishield evidence-pack
```

`watermark-check` 가 내 워터마크 ID를 찾아 레지스트리와 일치하면, 그 페이지는 내 원본에서 **그대로 복제**된 것입니다. `evidence-pack` 은 이를 입증할 자료(어떤 ID를 언제 박았는지, 산출물 해시, git 저작 이력, OpenTimestamps 시점증명, DMCA 통지서 초안)를 한 폴더로 묶습니다.

## 안전장치

- **백업·복원**: `apply` 는 변경 전 원본을 `.aishield-backup/` 에 보존하고, 새로 만든 파일도 추적합니다. `aishield restore` 로 정확히 원복됩니다(증거 레지스트리는 보존).
- **멱등성**: 같은 산출물에 두 번 적용해도 워터마크·고지가 중복으로 쌓이지 않습니다.
- **기능 게이트(F)**: 적용이 앱을 깼는지 정적 검증하고, 깨졌으면 즉시 `restore` 를 권합니다.
- **공격적 레이어 기본 OFF**: SEO·법 위험 레이어(G)는 `aggressiveOptIn: true` 없이는 켜지지 않습니다.

## 정직한 한계 (반드시 읽어주세요)

- **100% 차단은 불가능합니다.** 렌더하려면 런타임에 코드가 평문으로 존재합니다.
- **난독화는 우회됩니다.** 2025년 LLM 기반 디오퍼스케이터가 문자열 난독화를 상당 부분 복원합니다. "시간 벌기"입니다.
- **서버 이전도 블랙박스 추론에 취약합니다.** 입출력을 충분히 모으면 알고리즘을 근사할 수 있습니다.
- **봇은 robots.txt를 무시할 수 있습니다.** 준수 의지가 있는 크롤러만 걸러집니다.
- **워터마크는 재작성 복제에 무력합니다.** AI가 기능을 이해하고 처음부터 다시 짜면 어떤 워터마크도 전달되지 않습니다.
- **AI 고지의 효과는 불확실합니다.** 모델·AI 제공사 정책에 종속됩니다.
- **법적 효력은 관할권별로 다릅니다.** 이 도구는 법률 자문을 대체하지 않습니다.

이 한계를 알고도 쓰는 이유: **기회주의적 복제(그대로 긁어 붙이기)를 크게 줄이고, 복제가 일어나면 추적·입증·대응할 수단을 갖기 위해서**입니다.

## 자주 묻는 질문

**Q. AI가 아예 못 베끼게 되나요?**
아니요. 위 "정직한 한계"를 보세요. 비용을 크게 올리고, 그대로 베끼면 걸리게 하고, 증거를 남깁니다.

**Q. 사이트 기능이 깨지지 않나요?**
워터마크·고지·법적 레이어는 HTML 텍스트 노드와 메타만 건드리고 JS 로직은 바꾸지 않습니다. 난독화(옵션)만 JS를 바꿉니다. 적용 후 `aishield verify` 로 확인하고, 문제가 있으면 `aishield restore` 로 원복합니다.

**Q. 어떤 스택을 지원하나요?**
빌드 산출물이 있으면 스택을 가리지 않습니다. 다만 가장 강한 방어(로직 격리)는 서버가 있는 스택(Next.js 등)에서 더 효과적입니다.

**Q. 내 알림 채널/서버가 필요한가요?**
아니요. 이 도구는 외부 서버나 알림 인프라가 필요 없습니다. 전부 빌드 산출물 후처리와 로컬 파일로 동작합니다.
